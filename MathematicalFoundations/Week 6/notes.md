# Week 6: Theoretical Notes — Linear Models and Bias–Variance Analysis

## Overview

Week 6 is the **grand unification** of machine learning. We finally prove that the probabilistic approach (estimating distributions via divergence minimization) and the practical risk minimization approach (fitting a function to minimize error) are **mathematically identical** under the right assumptions. We introduce the fundamental machinery of modern ML: linear models solved via the Normal Equation or Gradient Descent, activation functions that enable classification, and the bias-variance tradeoff that explains why models fail on unseen data.

---

## The Two Philosophies: Probabilistic vs Risk Minimization

Machine learning can be viewed through two distinct lenses:

**The Probabilistic Approach:**
- Goal: Estimate the underlying probability distributions ($P_x, P_{xy}, P_{y|x}$) that generated the data.
- Method: Propose a model with parameters $\theta$, then minimize divergence (e.g., KL Divergence) between the proposed distribution $p_\theta$ and the true data distribution $P$.
- Foundation: Information theory and Bayesian statistics.
- Example: Fit a Gaussian $\mathcal{N}(\mu, \sigma^2)$ to data by minimizing KL divergence.

**The Risk Minimization Approach (Empirical Risk Minimization):**
- Goal: Find a function $h: X \rightarrow Y$ that makes the fewest predictions errors on training data.
- Method: Choose a loss function $L(h(x), y)$, then minimize the empirical risk: $\hat{R}(\theta) = \frac{1}{n} \sum_{i=1}^{n} L(h_\theta(x_i), y_i)$.
- Foundation: Statistical learning theory.
- Example: Fit a line by minimizing squared errors $\sum (h(x) - y)^2$.

**The Grand Revelation:** These are not competing philosophies. They are **mathematically equivalent**. When you assume Gaussian errors in the data, the probabilistic Maximum Likelihood Estimation becomes identical to minimizing Squared Error Loss in the risk minimization framework.

## Equivalence: Gaussian Likelihood = Squared Error Loss

Assume the data is generated by:
$$y = h_\theta(x) + \epsilon, \quad \epsilon \sim \mathcal{N}(0, I)$$

That is, the target $y$ is the model's prediction plus Gaussian noise with identity covariance.

**Probabilistic View - Maximum Likelihood:**

The likelihood of observing data $y_i$ given $x_i$ is:
$$p_\theta(y_i | x_i) = \mathcal{N}(y_i; h_\theta(x_i), I) = \frac{1}{(2\pi)^{d/2}} \exp\left(-\frac{1}{2} \|y_i - h_\theta(x_i)\|_2^2\right)$$

The log-likelihood for all $n$ samples:
$$\log L(\theta) = \sum_{i=1}^{n} \log p_\theta(y_i | x_i) = \text{constant} - \frac{1}{2} \sum_{i=1}^{n} \|y_i - h_\theta(x_i)\|_2^2$$

To maximize $\log L(\theta)$, we minimize $\sum_{i=1}^{n} \|y_i - h_\theta(x_i)\|_2^2$.

**Risk Minimization View:**

Minimize the empirical risk with squared error loss:
$$\hat{R}(\theta) = \frac{1}{n} \sum_{i=1}^{n} (y_i - h_\theta(x_i))^2$$

**They are identical!** (up to constant scaling). Maximizing likelihood under Gaussian assumptions = Minimizing squared error. This unification justifies why squared error loss is so fundamental.

Other distributional assumptions lead to other losses:
- **Laplace distribution** ($\epsilon \sim \text{Laplace}(0, b)$) → **Absolute Error Loss**: $|y - h(x)|$
- **Bernoulli distribution** (classification targets) → **Cross-Entropy Loss**: $-\sum_k y_k \log h_\theta(x)$

## Ordinary Least Squares (OLS)

The simplest linear model assumes:
$$h_\theta(x) = \theta^T x + \theta_0$$

where:
- $\theta \in \mathbb{R}^d$ are the weights (slopes for each feature).
- $\theta_0 \in \mathbb{R}$ is the bias/intercept.
- $x \in \mathbb{R}^d$ is the input feature vector.

### The Bias Trick

Writing $\theta^T x + \theta_0$ in formulas is cumbersome for matrix math. We use the **"Bias Trick"**: artificially append a 1 to the input:

$$X = \begin{bmatrix} x_1 \\ x_2 \\ \vdots \\ x_d \\ 1 \end{bmatrix} \in \mathbb{R}^{d+1}, \quad \theta = \begin{bmatrix} \theta_1 \\ \theta_2 \\ \vdots \\ \theta_d \\ \theta_0 \end{bmatrix} \in \mathbb{R}^{d+1}$$

Now:
$$h_\theta(x) = \theta^T X = \sum_{i=1}^{d} \theta_i x_i + \theta_0 \cdot 1 = \theta^T x + \theta_0$$

**Benefit:** All calculations become pure linear algebra using a single dot product.

### The Normal Equation

Given a dataset $\{(x_i, y_i)\}_{i=1}^{n}$, organize data into matrix form:
- Design matrix: $X \in \mathbb{R}^{n \times (d+1)}$ (each row is one augmented sample)
- Target vector: $\mathbf{y} \in \mathbb{R}^{n}$

The empirical risk with squared error:
$$\hat{R}(\theta) = \frac{1}{n} \|\mathbf{y} - X\theta\|_2^2$$

To find the minimum, take the gradient with respect to $\theta$:
$$\frac{\partial \hat{R}}{\partial \theta} = \frac{2}{n} X^T (X\theta - \mathbf{y}) = 0$$

Solving for $\theta$:
$$X^T X \theta = X^T \mathbf{y}$$
$$\theta^* = (X^T X)^{-1} X^T \mathbf{y}$$

This is the **Normal Equation**—an exact, closed-form solution. No iteration needed!

**Computational Note:** Matrix inversion $(X^T X)^{-1}$ is numerically expensive ($O(d^3)$). For high-dimensional data, iterative methods (Gradient Descent) are preferred.

**Geometric Interpretation:** $\theta^*$ is the orthogonal projection of $\mathbf{y}$ onto the column space of $X$. The residual $\mathbf{y} - X\theta^*$ is perpendicular to all columns of $X$.

## Generalized Linear Models (GLM/GLS)

What if the data is clearly non-linear (curved)? One approach: abandon linear models. 

**Better approach:** Keep the linear solver but **bend the data** using a fixed transformation $\phi(x)$.

### Feature Transformation

Instead of using raw features $x \in \mathbb{R}^d$, map them to a higher-dimensional space:
$$\phi(x): \mathbb{R}^d \rightarrow \mathbb{R}^m, \quad m > d$$

Then fit a linear model in the transformed space:
$$h_\theta(x) = \theta^T \phi(x)$$

**Examples of $\phi(x)$:**

1. **Polynomial Expansion** (univariate):
   $$\phi(x) = [1, x, x^2, x^3, \ldots, x^p]$$
   The model becomes: $h(x) = \theta_0 + \theta_1 x + \theta_2 x^2 + \cdots + \theta_p x^p$ (a polynomial of degree $p$).

2. **Radial Basis Functions (RBF):**
   $$\phi(x) = [\exp(-\gamma \|x - c_1\|^2), \exp(-\gamma \|x - c_2\|^2), \ldots]$$
   where $c_1, c_2, \ldots$ are fixed centers. This creates localized "bumps."

3. **Fourier Expansion:**
   $$\phi(x) = [\cos(x), \sin(x), \cos(2x), \sin(2x), \ldots]$$

### Why It Works

Once you have $\Phi = [\phi(x_1)^T; \phi(x_2)^T; \ldots; \phi(x_n)^T] \in \mathbb{R}^{n \times m}$, you **apply the Normal Equation as usual**:
$$\theta^* = (\Phi^T \Phi)^{-1} \Phi^T \mathbf{y}$$

The model is linear in the transformed space, but non-linear in the original space!

**Trade-off:** Higher-dimensional $\phi$ (more expressivity) requires more data to avoid overfitting. This is the bias-variance tradeoff, discussed later.

## Linear Models for Classification

Regression outputs any real number. Classification needs probabilities in $[0, 1]$. We introduce **activation functions** to squish the linear output.

### Sigmoid Function (Binary Classification)

For binary classification (two classes), use the **Sigmoid** function:
$$\sigma(t) = \frac{1}{1 + e^{-t}}$$

**Properties:**
- Domain: $\mathbb{R}$ (all real numbers)
- Range: $(0, 1)$ (strictly between 0 and 1)
- Monotonically increasing
- Smooth and differentiable everywhere (crucial for gradient descent)

**Model:**
$$P(y=1 | x) = \sigma(\theta^T x + \theta_0)$$

Interpretation: The probability that the sample belongs to class 1 is the sigmoid of the linear score.

**Intuition:** 
- If $\theta^T x + \theta_0 > 0$ (positive linear score), $\sigma > 0.5$ → predict class 1.
- If $\theta^T x + \theta_0 < 0$ (negative linear score), $\sigma < 0.5$ → predict class 0.
- The sigmoid smoothly transitions between classes rather than abruptly switching at a threshold.

**Mathematical Form:**
$$\sigma(t) = \frac{e^t}{1 + e^t}, \quad \frac{d\sigma}{dt} = \sigma(t)(1 - \sigma(t))$$

The derivative is beautiful: it depends only on the output value, not on the input.

### Softmax Function (Multiclass Classification)

For $K > 2$ classes, generalize sigmoid to **Softmax**:
$$\text{softmax}(\mathbf{z})_j = \frac{e^{z_j}}{\sum_{k=1}^{K} e^{z_k}}, \quad j = 1, \ldots, K$$

where $\mathbf{z} = [z_1, \ldots, z_K]^T$ are the $K$ linear scores.

**Properties:**
- Each output $Z_j \in (0, 1)$ (a probability).
- All outputs sum to 1: $\sum_{j=1}^{K} Z_j = 1$.
- Differentiable and stable (exponentials don't overflow due to normalization).

**Model:**
$$P(y = j | x) = \text{softmax}(\theta_j^T x)_j = \frac{e^{\theta_j^T x}}{\sum_{k=1}^{K} e^{\theta_k^T x}}$$

Each class has its own weight vector $\theta_j$, organized as a matrix $\Theta \in \mathbb{R}^{d \times K}$.

**Interpretation:**
- The linear scores $[\theta_1^T x, \theta_2^T x, \ldots, \theta_K^T x]$ reflect the "raw preference" for each class.
- Softmax converts these into a normalized probability distribution.
- The highest score maps to the highest probability, but softmax allows soft assignments (uncertainty).

## Gradient Descent Algorithm

The Normal Equation $(X^T X)^{-1} X^T \mathbf{y}$ only works for models where the loss function is quadratic (like squared error with linear models). Once we introduce Sigmoid, Softmax, or other non-linear activation functions, the loss becomes non-convex or at least hard to solve analytically.

### Algorithm

**Initialization:** Start with random $\theta^{(0)}$.

**Iteration:** For $t = 0, 1, 2, \ldots$:
$$\theta^{(t+1)} = \theta^{(t)} - \alpha \nabla_\theta \hat{R}(\theta^{(t)})$$

where:
- $\alpha > 0$ is the **learning rate** (step size).
- $\nabla_\theta \hat{R}(\theta)$ is the gradient of the loss with respect to $\theta$.

**Stopping:** When the loss stops decreasing significantly or after a fixed number of iterations.

### Intuition

Imagine being blindfolded on a mountain, trying to reach the valley (minimum error):
- You feel the ground's slope under your feet ($\nabla \hat{R}$).
- You take a small step **downhill** (direction of steepest descent, hence the minus sign).
- You repeat until you reach a local minimum.

### Learning Rate $\alpha$

- **Too large:** You overshoot the minimum, oscillate, or diverge.
- **Too small:** You move slowly, requiring many iterations and slow convergence.
- **Adaptive schemes:** Adam, RMSprop adjust $\alpha$ per iteration for faster convergence.

### Convergence

For **convex** loss functions (like squared error), gradient descent converges to the global optimum. For **non-convex** functions (like neural networks), it converges to a local optimum.

## Cross-Entropy Loss

For classification, we do **not** use squared error loss. Instead, we use **Cross-Entropy Loss**, which heavily penalizes confident wrong predictions.

### Binary Cross-Entropy

For binary classification with predicted probability $\hat{p} = \sigma(\theta^T x) \in (0, 1)$ and true label $y \in \{0, 1\}$:
$$L_C(y, \hat{p}) = -[y \log \hat{p} + (1-y) \log(1 - \hat{p})]$$

**Interpretation:**
- If $y = 1$: $L_C = -\log \hat{p}$. Minimizing this means maximizing $\hat{p}$ (the probability of class 1).
- If $y = 0$: $L_C = -\log(1 - \hat{p})$. Minimizing this means minimizing $\hat{p}$.

**Why not squared error?** Squared error treats a prediction of 0.1 (when $y=1$) the same as 0.9 (when $y=1$) relative to the same absolute error. Cross-entropy is asymmetric: it heavily penalizes extreme confidence in the wrong direction.

### Multiclass Cross-Entropy

For $K$-class classification with one-hot encoded target $\mathbf{y} = [y_1, \ldots, y_K]$ (exactly one $y_j = 1$) and predicted probabilities $\hat{\mathbf{p}} = \text{softmax}(\theta^T x)$:
$$L_C(\mathbf{y}, \hat{\mathbf{p}}) = -\sum_{j=1}^{K} y_j \log \hat{p}_j$$

Only the term with $y_j = 1$ survives: $L_C = -\log \hat{p}_{true}$. Minimizing this maximizes the probability of the true class.

### Connection to Information Theory

Cross-entropy is defined as:
$$H(P, Q) = -\sum_i P(i) \log Q(i)$$

It measures how much information is needed to describe the true distribution $P$ using a model $Q$. Minimizing cross-entropy is equivalent to minimizing the KL divergence from $P$ to $Q$ (up to the constant entropy of $P$).

## Bias–Variance Decomposition

The true goal of machine learning is **generalization**: how well will our model perform on **unseen test data**?

### Expected Risk Decomposition

Given a fixed true model $h^*: X \rightarrow Y$, a hypothesis space $\mathcal{H}$ of candidate models, and loss function $L$, the expected error on a test sample $(x, y)$ decomposes as:
$$\mathbb{E}[\text{Error}] = \text{Bias}^2 + \text{Variance} + \text{Irreducible Noise}$$

where the expectation is over all possible training datasets $D_{\text{train}}$.

### Bias (Underfitting)

$$\text{Bias} = \mathbb{E}_D[\hat{h}(x)] - h^*(x)$$

This is the systematic error from choosing a **too-simple model**. 

**Example:** Fitting a straight line to data that follows a parabola always leaves systematic residuals. No matter how much training data you get, the line can never curve. Bias persists.

**Intuition:** The model is "stubborn"—it refuses to adapt to the true function's complexity.

### Variance (Overfitting)

$$\text{Variance} = \mathbb{E}_D[(\hat{h}(x) - \mathbb{E}_D[\hat{h}(x)])^2]$$

This measures how much the learned model **changes if you use a different training dataset**.

**Example:** A very high-degree polynomial (degree 100) can pass through every data point exactly. If you change one training sample, the polynomial wiggles dramatically at other points. High variance.

**Intuition:** The model is "anxious"—it reacts strongly to small changes in data.

### Irreducible Noise

$$\text{Irreducible Noise} = \mathbb{E}[(h^*(x) - y)^2 | x]$$

The universe is inherently noisy. Some label variation is due to randomness, not the input features. Even a perfect model cannot reduce this.

### The Trade-off

- **Simple models** (e.g., low-degree polynomials): High bias, low variance.
- **Complex models** (e.g., high-degree polynomials): Low bias, high variance.
- **Goldilocks zone:** The optimal model balances the two.

**Practical Tip:** Modern practitioners often start with a highly complex model (in the overfitting region) and then regularize (add penalties) or select features to dial back variance while keeping bias manageable.

### Visualizing the Trade-off

For varying model complexity $\lambda$ (from simple to complex):
- Training error: Always decreases (complex models fit training data better).
- Test error: Decreases then increases (at first, lowering bias helps; later, rising variance hurts).
- U-shaped curve: The bottom is the "sweet spot."

## Summary

**Linear models** are the foundation of modern machine learning:

1. **Unification:** The probabilistic approach (Maximum Likelihood) and risk minimization (ERM) are mathematically equivalent under Gaussian assumptions.

2. **OLS:** Linear regression with squared error loss has a closed-form solution via the Normal Equation. The "bias trick" elegantly incorporates the intercept.

3. **Non-linearity via Transformation:** Generalized Linear Models achieve non-linear expressivity by applying fixed transformations $\phi(x)$ before fitting a linear model.

4. **Classification:** Sigmoid (binary) and Softmax (multiclass) activation functions map linear scores to probabilities.

5. **Gradient Descent:** When analytical solutions fail, gradient descent iteratively updates parameters by following the negative gradient of the loss.

6. **Cross-Entropy Loss:** For classification, cross-entropy loss (derived from information theory) outperforms squared error by heavily penalizing confident wrong predictions.

7. **Generalization:** The bias-variance decomposition explains why models fail on test data. Optimal generalization requires balancing bias (underfitting) and variance (overfitting).

| Concept | Definition | Example |
|---------|-----------|---------|
| **Probabilistic Approach** | Estimate probability distributions via divergence minimization | MLE with Gaussian assumption leads to squared error loss |
| **Risk Minimization (ERM)** | Minimize empirical loss on training data | Linear regression: minimize $\frac{1}{n}\sum (y_i - \hat{y}_i)^2$ |
| **Normal Equation** | Closed-form solution for linear least squares: $\theta^* = (X^T X)^{-1} X^T \mathbf{y}$ | Solves OLS in one step without iteration |
| **Bias Trick** | Append 1 to input and bias to $\theta$ to unify intercept into one dot product | Convert $\theta^T x + \theta_0$ to $\theta^T[x; 1]$ |
| **Feature Transformation** | Apply fixed $\phi(x)$ to enable non-linear models while keeping linear solver | Polynomial: $\phi(x) = [1, x, x^2, x^3]$; model is linear in transformed space |
| **Sigmoid Function** | $\sigma(t) = \frac{1}{1+e^{-t}}$; maps $\mathbb{R} \to (0,1)$; used for binary classification | Logistic regression: $P(y=1|x) = \sigma(\theta^T x)$ |
| **Softmax Function** | Normalize multiple scores to valid probability distribution: $\text{softmax}(z)_j = \frac{e^{z_j}}{\sum_k e^{z_k}}$ | Multiclass classifier with K output probabilities summing to 1 |
| **Gradient Descent** | Iterative update: $\theta^{(t+1)} = \theta^{(t)} - \alpha \nabla \hat{R}(\theta^{(t)})$ | Used when Normal Equation unavailable (non-linear losses) |
| **Cross-Entropy Loss** | $L_C = -\sum_j y_j \log \hat{p}_j$; heavily penalizes confident wrong predictions | Standard loss for classification; derived from information theory |
| **Bias–Variance Trade-off** | Error = Bias² (underfitting) + Variance (overfitting) + Noise; optimal model balances both | Simple model: high bias, low variance; complex model: low bias, high variance |
| **Generalization** | Goal is to minimize test error on unseen data, not just training error | Model that memorizes training set (0% train error) may fail on test set |
