## Question Set 1: Two Philosophies of Machine Learning

### Q1 (1 point)

**In the context of the probabilistic approach versus the risk minimization approach, which of the following statements is true regarding their objectives?**

- [ ] A) The probabilistic approach estimates a direct function $h:X \rightarrow Y$ via Empirical Risk Minimization.
- [ ] B) The risk minimization approach aims to estimate distributions $P_x$ & $P_{y|x}$ via divergence minimization.
- [x] C) The probabilistic approach estimates distributions such as $P_{y|x}$ via divergence minimization, while the risk minimization approach estimates a function $h$ from the dataset.
- [ ] D) Both approaches solve for the same objective function $(X^T X)^{-1} X^T \mathbf{y}$ regardless of the model type.

**Answer:** C

**Answer Explanation:**

This question tests understanding of the **two foundational philosophies** in machine learning.

**The Probabilistic Approach:**
- **Goal:** Understand the entire data generation process by estimating probability distributions.
- **Method:** Model the data using parameterized distributions (e.g., $p_\theta(y|x) = \mathcal{N}(h_\theta(x), \sigma^2)$), then find parameters by **minimizing divergence** (like KL Divergence) between the proposed distribution and the true data distribution.
- **Questions it answers:** "What is the shape of the underlying probability distribution?"
- **Perspective:** Inference-oriented; answers "what are the probabilities?"

**The Risk Minimization Approach (ERM):**
- **Goal:** Find the best mapping function from inputs to outputs.
- **Method:** Define a loss function $L(h(x), y)$ and minimize empirical risk: $\hat{R}(\theta) = \frac{1}{n} \sum_{i=1}^{n} L(h_\theta(x_i), y_i)$.
- **Questions it answers:** "What function best predicts targets?"
- **Perspective:** Optimization-oriented; answers "what is the best predictor?"

**Analysis of Incorrect Options:**

- **A)** Is exactly **backward**. It describes the Risk Minimization approach (direct function via ERM), not the probabilistic approach.
- **B)** Is also **backward**. It describes the Probabilistic approach, not Risk Minimization.
- **D)** The formula $(X^T X)^{-1} X^T \mathbf{y}$ is specific to **linear regression** (OLS). It does not apply to all model types (e.g., logistic regression, neural networks, decision trees). This is factually incorrect.

**Key Insight:** Week 6's central theme is that these two philosophies are **mathematically equivalent** when you assume Gaussian errors. This unification justifies both perspectives.

**Related Topics:** [The Two Philosophies: Probabilistic vs Risk Minimization](notes.md#the-two-philosophies-probabilistic-vs-risk-minimization)

### Q2 (1 point)

**It is claimed that Divergence Minimization and ERM are equivalent in certain scenarios. For a model where $p_\theta(y|x) \sim \mathcal{N}(y; h_\theta(x), I)$, which loss function in ERM makes the Maximum Likelihood objective equivalent to ERM?**

- [ ] A) Cross-Entropy Loss
- [x] B) Squared Error Loss
- [ ] C) Hinge Loss
- [ ] D) Absolute Error Loss

**Answer:** B

**Answer Explanation:**

This is the **mathematical proof** that the two philosophies are equivalent under Gaussian assumptions.

**Probabilistic Setup:**

Assume the data is generated by:
$$y = h_\theta(x) + \epsilon, \quad \epsilon \sim \mathcal{N}(0, I)$$

The conditional likelihood for one sample:
$$p_\theta(y|x) = \frac{1}{(2\pi)^{d/2}} \exp\left(-\frac{1}{2} \|y - h_\theta(x)\|_2^2\right)$$

**Maximum Likelihood Objective:**

For all $n$ samples:
$$\log L(\theta) = \sum_{i=1}^{n} \log p_\theta(y_i|x_i) = \text{const} - \frac{1}{2} \sum_{i=1}^{n} \|y_i - h_\theta(x_i)\|_2^2$$

To **maximize** $\log L(\theta)$, you **minimize** $\sum_{i=1}^{n} \|y_i - h_\theta(x_i)\|_2^2$.

**Risk Minimization Objective:**

Empirical risk with squared error loss:
$$\hat{R}(\theta) = \frac{1}{n} \sum_{i=1}^{n} (y_i - h_\theta(x_i))^2$$

**These are identical!** (up to scaling by $1/n$ and constant $1/2$). The exponential and logarithm in the Gaussian formula cancel, leaving only the squared error term.

**Why Other Distributions Lead to Different Losses:**

- **Laplace Distribution** ($\epsilon \sim \text{Laplace}(0, b)$): Likelihood contains $e^{-|y-h|/b}$ → Minimizing MLE leads to **Absolute Error Loss**: $|y - h|$.
- **Bernoulli Distribution** (classification): Likelihood contains $y \log p + (1-y) \log(1-p)$ → Leads to **Cross-Entropy Loss**.

**Analysis of Incorrect Options:**

- **A) Cross-Entropy Loss:** Used for classification (Bernoulli targets), not regression with continuous Gaussian targets.
- **C) Hinge Loss:** $\max(0, 1 - yh)$ is specific to **Support Vector Machines** and doesn't connect to Gaussian likelihoods.
- **D) Absolute Error Loss:** Corresponds to Laplace errors, not Gaussian errors.

**Key Insight:** This equivalence is profound. It says: "To maximize the likelihood of Gaussian-distributed data, just minimize squared errors—no need to think about probabilities!"

**Related Topics:** [Equivalence: Gaussian Likelihood = Squared Error Loss](notes.md#equivalence-gaussian-likelihood--squared-error-loss)

## Question Set 2: Ordinary Least Squares and the Normal Equation

### Q3 (1 point)

**To represent a linear model $h_\theta(x) = \theta^T x + \theta_0$ as a single dot product $h_\theta(x) = \theta^T X$, how must the input vector $x \in \mathbb{R}^d$ and the parameter vector $\theta$ be transformed?**

- [ ] A) $X = [x, 0]^T$ & $\theta \in \mathbb{R}^d$
- [x] B) $X = [x, 1]^T$ & $\theta \in \mathbb{R}^{d+1}$
- [ ] C) $X = \phi(x)$ & $\theta \in \mathbb{R}^d$
- [ ] D) No transformation is needed; the bias is always zero in ERM.

**Answer:** B

**Answer Explanation:**

This question introduces the **"Bias Trick"**, a fundamental notational convenience in machine learning.

**The Problem:**

The standard linear model has two parts:
$$h_\theta(x) = \theta^T x + \theta_0 = \underbrace{\theta_1 x_1 + \cdots + \theta_d x_d}_{\text{weights}} + \underbrace{\theta_0}_{\text{bias}}$$

Writing formulas with a separate $+ \theta_0$ term is cumbersome in matrix notation. We want a single dot product.

**The Solution: The Bias Trick**

Artificially append a 1 to the input vector and move the bias to the weight vector:

$$X = \begin{bmatrix} x_1 \\ x_2 \\ \vdots \\ x_d \\ 1 \end{bmatrix} \in \mathbb{R}^{d+1}, \quad \theta = \begin{bmatrix} \theta_1 \\ \theta_2 \\ \vdots \\ \theta_d \\ \theta_0 \end{bmatrix} \in \mathbb{R}^{d+1}$$

Now:
$$\theta^T X = \theta_1 x_1 + \cdots + \theta_d x_d + \theta_0 \cdot 1 = \theta^T x + \theta_0$$

**Why the 1?**

Multiplying anything by 1 gives the identity. So $\theta_0 \cdot 1 = \theta_0$ automatically without extra notation. If we used $X = [x, 0]^T$, the bias would disappear: $\theta_0 \cdot 0 = 0$ (wrong!).

**Benefits:**

1. All formulas become pure linear algebra: $\theta^T X$ (no exceptions).
2. Matrix operations (inversion, multiplication) apply uniformly.
3. Code is cleaner: one function that handles both weights and bias.

**Analysis of Incorrect Options:**

- **A)** Uses 0 instead of 1, causing the bias to vanish entirely.
- **C)** Refers to feature transformation $\phi(x)$ (a different concept for non-linear models), not the bias trick.
- **D)** The bias $\theta_0$ is almost never zero. A model forced to pass through the origin (0 → 0) is rarely useful.

**Practical Note:** In code, libraries often add the bias column implicitly or track it separately, but mathematically, the bias trick is foundational.

**Related Topics:** [Ordinary Least Squares (OLS)](notes.md#ordinary-least-squares-ols)

### Q4 (1 point)

**For a linear model with squared error loss, what is the analytical solution for the optimal parameters $\theta^*$?**

- [ ] A) $\theta^* = \theta - \alpha \nabla \hat{R}(\theta)$
- [x] B) $\theta^* = (X^T X)^{-1} X^T \mathbf{y}$
- [ ] C) $\theta^* = \sigma(\theta^T x)$
- [ ] D) $\theta^* = \text{softmax}(\theta^T x)$

**Answer:** B

**Answer Explanation:**

This is the **Normal Equation**, the closed-form solution for linear least squares regression.

**Problem Setup:**

Given $n$ training samples and empirical risk:
$$\hat{R}(\theta) = \frac{1}{n} \|\mathbf{y} - X\theta\|_2^2$$

We want to find $\theta$ that minimizes this.

**Derivation:**

Since squared error is differentiable, take the gradient with respect to $\theta$ and set it to zero:
$$\frac{\partial \hat{R}}{\partial \theta} = \frac{2}{n} X^T(X\theta - \mathbf{y}) = 0$$

Rearrange:
$$X^T X \theta = X^T \mathbf{y}$$

Assuming $(X^T X)$ is invertible (full rank):
$$\theta^* = (X^T X)^{-1} X^T \mathbf{y}$$

**Interpretation:**

- $X^T X \in \mathbb{R}^{(d+1) \times (d+1)}$: Measures feature co-variance.
- $(X^T X)^{-1}$: "Inverts" the feature correlations.
- $X^T \mathbf{y}$: Correlates features with targets.

**Geometric Meaning:** $\theta^*$ is the **orthogonal projection** of the target vector $\mathbf{y}$ onto the column space of $X$. The residual $\mathbf{y} - X\theta^*$ is perpendicular to all columns of $X$.

**Computational Complexity:**

- Matrix inversion: $O(d^3)$.
- For large $d$ (thousands of features), this is slow.
- Modern systems use iterative solvers (Gradient Descent) instead.

**Analysis of Incorrect Options:**

- **A)** $\theta^* = \theta - \alpha \nabla \hat{R}(\theta)$ is the **Gradient Descent update rule**. It's iterative, not a closed-form solution. Used when the Normal Equation is unavailable (e.g., logistic regression).
- **C)** $\theta^* = \sigma(\theta^T x)$ is the **output** of a logistic regression model, not the solution for parameters.
- **D)** $\theta^* = \text{softmax}(\theta^T x)$ is the **output** of a multiclass classifier, not how you solve for $\theta$.

**Key Insight:** The Normal Equation is a "magic formula"—it solves linear regression in one shot without iteration. This only works because squared error is convex and has a closed-form minimum.

**Related Topics:** [Ordinary Least Squares (OLS)](notes.md#ordinary-least-squares-ols)

## Question Set 3: Generalizing Linear Models

### Q5 (1 point)

**In Generalized Linear Models, how can a user achieve non-linear combinations of data dimensions while still using a linear solver?**

- [ ] A) By using a non-linear loss function like Cross-Entropy.
- [x] B) By applying a fixed transformation $\phi(x)$ to the data before performing ERM.
- [ ] C) By iteratively updating $\theta$ using second-order gradients.
- [ ] D) By increasing the dimensionality of the target $\mathbf{y}$.

**Answer:** B

**Answer Explanation:**

This is the key idea behind **Generalized Linear Models (GLM)**. We keep the linear solver but bend the data to enable non-linear expressivity.

**The Problem:**

A straight line cannot fit curved data. If your data follows a parabola $y = x^2$, no line will fit well.

**The Solution: Feature Transformation**

Instead of using raw features $x$, transform them to a higher-dimensional space via a fixed function $\phi: \mathbb{R}^d \rightarrow \mathbb{R}^m$ (usually $m > d$):

$$h_\theta(x) = \theta^T \phi(x)$$

Then fit a linear model in the transformed space using the Normal Equation with $\Phi$ (the design matrix of transformed features) instead of $X$:
$$\theta^* = (\Phi^T \Phi)^{-1} \Phi^T \mathbf{y}$$

**Examples:**

1. **Polynomial Features** (univariate):
   $$\phi(x) = [1, x, x^2, x^3, \ldots, x^p]$$
   The model becomes a polynomial: $h(x) = \theta_0 + \theta_1 x + \theta_2 x^2 + \cdots + \theta_p x^p$.

2. **Radial Basis Functions (RBF):**
   $$\phi(x) = [\exp(-\gamma \|x - c_1\|^2), \exp(-\gamma \|x - c_2\|^2), \ldots]$$
   Each feature measures distance from a fixed center. Creates smooth, localized "bumps."

3. **Fourier Features:**
   $$\phi(x) = [\sin(\omega_1 x), \cos(\omega_1 x), \sin(\omega_2 x), \ldots]$$
   Decomposes the function into oscillating components. Useful for periodic data.

**Why It Works:**

The linear solver $(X^T X)^{-1} X^T \mathbf{y}$ doesn't care about the **source** of the features. It treats $\phi(x)$ exactly like any other input. The model is **linear in the transformed space** but **non-linear in the original space**.

**Trade-offs:**

- **More complex $\phi$** (higher-dimensional $m$): Better fits curved data (lower bias).
- **Downside:** Requires more training data to avoid overfitting (higher variance). This is the **bias-variance tradeoff**.

**Analysis of Incorrect Options:**

- **A)** A non-linear loss function (like cross-entropy) doesn't enable the linear solver to fit curves. Loss functions measure error, not expressivity.
- **C)** Second-order gradients (Newton's method) make optimization faster but don't enable non-linearity in the model itself.
- **D)** Increasing the dimensionality of the target $\mathbf{y}$ (e.g., from scalar to vector) doesn't help fit a non-linear function. We want to change the input space, not the output.

**Key Insight:** "We don't bend our model; we bend the data." This is conceptually elegant and computationally efficient.

**Related Topics:** [Generalized Linear Models (GLM/GLS)](notes.md#generalized-linear-models-glmgls)

## Question Set 4: Classification with Activation Functions

### Q6 (1 point)

**Why is the sigmoid function $\sigma(t) = 1/(1+e^{-t})$ used in binary classification models like Logistic Regression?**

- [ ] A) To ensure the output is always a one-hot vector.
- [x] B) To map the real-valued output of $\theta^T x$ to the [0,1] range, representing a probability.
- [ ] C) To calculate the inverse of the matrix $X^T X$.
- [ ] D) To eliminate the need for a bias term $\theta_0$.

**Answer:** B

**Answer Explanation:**

The **Sigmoid function** is the bridge between linear models (which output real numbers) and binary classification (which requires probabilities in [0, 1]).

**The Problem:**

A linear model $\theta^T x$ outputs any value from $-\infty$ to $+\infty$:
- $\theta^T x = 100$ (very positive)
- $\theta^T x = -50$ (very negative)
- $\theta^T x = 0$ (neutral)

But for classification ("Is this spam?" Yes/No), we need a **probability** between 0 and 1.

**The Solution: The Sigmoid Function**

$$\sigma(t) = \frac{1}{1 + e^{-t}} = \frac{e^t}{1 + e^t}$$

**Properties:**

| Property | Value |
|----------|-------|
| Domain | All real numbers $(-\infty, \infty)$ |
| Range | Strictly between 0 and 1: $(0, 1)$ |
| Limit as $t \to +\infty$ | $\sigma(t) \to 1$ |
| Limit as $t \to -\infty$ | $\sigma(t) \to 0$ |
| Value at $t = 0$ | $\sigma(0) = 0.5$ |
| Monotonicity | Strictly increasing |
| Smoothness | Infinitely differentiable |
| Derivative | $\frac{d\sigma}{dt} = \sigma(t)(1 - \sigma(t))$ |

**Logistic Regression Model:**

$$P(y = 1 | x) = \sigma(\theta^T x) = \frac{1}{1 + e^{-\theta^T x}}$$

Interpretation:
- If $\theta^T x > 0$ (positive score), $\sigma > 0.5$ → Predict class 1.
- If $\theta^T x < 0$ (negative score), $\sigma < 0.5$ → Predict class 0.
- If $\theta^T x = 0$ (neutral), $\sigma = 0.5$ → Maximum uncertainty.

**Graphical Intuition:** The sigmoid is an S-shaped curve. It smoothly transitions from 0 to 1, preventing abrupt "cliff" jumps at decision boundaries.

**Why Not Other Functions?**

- **Step function** ($y = 1$ if $x > 0$ else $0$): Not differentiable; gradient descent fails.
- **Linear function** ($y = x$): Not bounded to [0, 1]; outputs can be > 1.
- **Sigmoid:** Smooth, bounded, differentiable everywhere.

**Analysis of Incorrect Options:**

- **A)** A one-hot vector is a discrete representation (e.g., [1, 0, 0] for class 1 out of 3 classes). Sigmoid outputs a continuous probability (e.g., 0.73), not a discrete vector.
- **C)** Matrix inversion $(X^T X)^{-1}$ is used for solving linear regression (OLS). Sigmoid is for classification and doesn't involve matrix inversion.
- **D)** Sigmoid doesn't eliminate the bias. The bias $\theta_0$ is added to $\theta^T x$ **before** the sigmoid: $\sigma(\theta^T x + \theta_0)$. The sigmoid just "squishes" the result.

**Key Insight:** Sigmoid is one of the simplest activation functions but fundamental. Modern neural networks use ReLU and others, but sigmoid remains important for binary classification.

**Related Topics:** [Linear Models for Classification](notes.md#linear-models-for-classification)

### Q7 (1 point)

**For a K-class classification problem, what is the primary property of the output vector $Z$ produced by the Softmax function?**

- [ ] A) $Z_j$ is always either 0 or 1.
- [ ] B) The vector $Z$ is the gradient of the loss function.
- [x] C) $Z_j \in [0,1]$ & $\sum_j Z_j = 1$
- [ ] D) $Z = (X^T X)^{-1} X^T \mathbf{y}$

**Answer:** C

**Answer Explanation:**

The **Softmax function** is the multiclass generalization of sigmoid. It converts raw scores into a valid **probability distribution**.

**The Function:**

For raw scores $\mathbf{z} = [z_1, z_2, \ldots, z_K]^T \in \mathbb{R}^K$:

$$\text{softmax}(\mathbf{z})_j = \frac{e^{z_j}}{\sum_{k=1}^{K} e^{z_k}}, \quad j = 1, \ldots, K$$

**Output Properties:**

1. **Range:** Each output $Z_j = \text{softmax}(\mathbf{z})_j \in (0, 1)$ (strictly between 0 and 1).
2. **Sum:** All outputs sum to exactly 1: $\sum_{j=1}^{K} Z_j = 1$.

Why is this important? These two properties define a **valid probability distribution over the $K$ classes**.

**Mathematical Proof:**

The sum property follows directly from the definition:
$$\sum_{j=1}^{K} \text{softmax}(\mathbf{z})_j = \sum_{j=1}^{K} \frac{e^{z_j}}{\sum_{k=1}^{K} e^{z_k}} = \frac{\sum_{j=1}^{K} e^{z_j}}{\sum_{k=1}^{K} e^{z_k}} = \frac{\text{(sum of numerators)}}{\text{(denominator)}} = 1$$

**Multiclass Classification Model:**

Each class has its own weight vector $\theta_j \in \mathbb{R}^d$. Organize into a matrix $\Theta \in \mathbb{R}^{d \times K}$.

Raw scores for all classes:
$$\mathbf{z} = \Theta^T x$$

Predicted class probabilities:
$$P(y = j | x) = \text{softmax}(\Theta^T x)_j = \frac{e^{\theta_j^T x}}{\sum_{k=1}^{K} e^{\theta_k^T x}}$$

**Example: 3-Class Classification**

Raw scores: $\mathbf{z} = [5, 2, 1]$ (strong preference for class 1).

Softmax output:
$$Z_1 = \frac{e^5}{e^5 + e^2 + e^1} = \frac{148.4}{156.2} \approx 0.95$$
$$Z_2 = \frac{e^2}{e^5 + e^2 + e^1} \approx 0.04$$
$$Z_3 = \frac{e^1}{e^5 + e^2 + e^1} \approx 0.01$$

Sum: $0.95 + 0.04 + 0.01 = 1.0$ ✓

The model is very confident in class 1 (95%), less confident in class 2 (4%), and almost certain class 3 is wrong (1%).

**Numerical Stability Tip:** Computing $e^{z_j}$ directly can overflow for large $z_j$. Use the stable trick: $\text{softmax}(\mathbf{z})_j = \text{softmax}(\mathbf{z} - \max(\mathbf{z}))_j$ (subtracts the max before exponentiating).

**Analysis of Incorrect Options:**

- **A)** "$Z_j$ is always either 0 or 1": This would mean the model is 100% confident in one class and 0% in all others. Softmax allows **soft assignments** (e.g., 0.95, 0.04, 0.01), which is more realistic and provides a confidence measure.
- **B)** "The vector $Z$ is the gradient of the loss function": The gradient measures how to update parameters. Softmax is the **prediction output** (the forward pass), not the gradient (backward pass).
- **D)** "$Z = (X^T X)^{-1} X^T \mathbf{y}$": This is the Normal Equation for linear regression, completely unrelated to softmax.

**Key Insight:** Softmax ensures the model outputs a valid probability distribution, which is essential for information-theoretic loss functions like cross-entropy.

**Related Topics:** [Linear Models for Classification](notes.md#linear-models-for-classification)

## Question Set 5: Optimization and Loss

### Q8 (1 point)

**When an optimization problem (like Logistic Regression) cannot be solved analytically, which numerical update rule is typically used?**

- [ ] A) $\theta_{t+1} = (X^T X)^{-1} X^T \mathbf{y}$
- [x] B) $\theta_{t+1} = \theta_t - \alpha \nabla_\theta \hat{R}(\theta_t)$
- [ ] C) $\theta_{t+1} = \text{softmax}(\theta_t)$
- [ ] D) $\theta_{t+1} = \theta_t + \alpha \nabla_\theta \hat{R}(\theta_t)$

**Answer:** B

**Answer Explanation:**

When the Normal Equation fails (because the loss is non-convex or non-quadratic), we use **Gradient Descent**, an iterative numerical optimization algorithm.

**Why the Normal Equation Fails:**

The Normal Equation $(X^T X)^{-1} X^T \mathbf{y}$ assumes:
1. The loss is **quadratic** (squared error).
2. The model is **linear** in parameters.

Once we introduce sigmoid, softmax, or other non-linear functions:
- The loss becomes non-quadratic.
- The cost surface has plateaus, valleys, and ridges.
- No closed-form solution exists.

**Gradient Descent Algorithm:**

**Initialization:** Start with random $\theta^{(0)}$.

**Iteration:** For $t = 0, 1, 2, \ldots$ (until convergence):
$$\theta_{t+1} = \theta_t - \alpha \nabla_\theta \hat{R}(\theta_t)$$

where:
- $\alpha > 0$: **Learning rate** (step size). Controls how far to move each iteration.
- $\nabla_\theta \hat{R}(\theta_t)$: **Gradient of empirical risk** at current parameters. Points in the direction of **steepest increase** in error.
- Minus sign: Move **opposite** the gradient (toward steepest **decrease**).

**Intuition: Blindfolded Mountain Descent**

Imagine you're blindfolded on a mountain, trying to reach the valley (minimum loss):
- You feel the slope of the ground under your feet ($\nabla \hat{R}$).
- You take a step **downhill** (opposite the slope), scaled by step size $\alpha$.
- You repeat until you reach a flat spot (local minimum).

**Learning Rate $\alpha$:**

| $\alpha$ Value | Effect |
|---|---|
| Too large (e.g., 1.0) | Overshoot the minimum; oscillate wildly; diverge |
| Too small (e.g., 0.00001) | Move very slowly; requires many iterations; slow convergence |
| Just right | Converge in reasonable time to near-optimal $\theta^*$ |

**Convergence Guarantees:**

- **Convex loss** (e.g., logistic regression, linear regression): Guaranteed to converge to global optimum.
- **Non-convex loss** (e.g., neural networks): Converges to a **local optimum** (may not be global best).

**Practical Variants:**

Modern deep learning uses adaptive methods:
- **Momentum:** Accumulate gradients over time for smoother descent.
- **Adam, RMSprop:** Adjust learning rate per parameter for faster convergence.

**Analysis of Incorrect Options:**

- **A)** $\theta_{t+1} = (X^T X)^{-1} X^T \mathbf{y}$: This is the analytical **closed-form solution** (Normal Equation). We use this **only if** we can solve analytically. This option contradicts the premise ("cannot be solved analytically").
- **C)** $\theta_{t+1} = \text{softmax}(\theta_t)$: Softmax normalizes probabilities; it doesn't update parameters. This makes no sense as an optimization step.
- **D)** Plus sign ($+$) instead of minus: $\theta_{t+1} = \theta_t + \alpha \nabla_\theta \hat{R}(\theta_t)$ moves **up** the gradient (toward higher error). This is **gradient ascent** (maximization), the opposite of what we want. Very wrong.

**Key Insight:** The minus sign is crucial. One of the most common mistakes in implementing gradient descent is forgetting the minus sign.

**Related Topics:** [Gradient Descent Algorithm](notes.md#gradient-descent-algorithm)

### Q9 (1 point)

**The Cross-Entropy loss for a classification model $h_\theta(x)$ and target $y$ is defined as:**

- [ ] A) $L_C = \|h_\theta(x) - y\|_2^2$
- [x] B) $L_C = -\sum_k y_k \log h_\theta(x)$
- [ ] C) $L_C = \sigma(\theta^T x)$
- [ ] D) $L_C = \max(0, 1 - y h_\theta(x))$

**Answer:** B

**Answer Explanation:**

**Cross-Entropy Loss** is the standard loss function for classification. It's derived from information theory and heavily penalizes confident wrong predictions.

**Definition:**

For $K$-class classification with one-hot encoded target $\mathbf{y} = [y_1, \ldots, y_K]$ (exactly one $y_j = 1$, others 0) and predicted probabilities $\hat{\mathbf{p}} = h_\theta(x) = \text{softmax}(\theta^T x)$:

$$L_C(\mathbf{y}, \hat{\mathbf{p}}) = -\sum_{j=1}^{K} y_j \log \hat{p}_j$$

**Simplification:**

Since only one $y_j = 1$ (say $y_i = 1$ for the true class $i$):
$$L_C = -1 \cdot \log \hat{p}_i - 0 \cdot \log \hat{p}_1 - 0 \cdot \log \hat{p}_2 - \cdots = -\log \hat{p}_i$$

The loss is just the **negative logarithm of the predicted probability of the true class**.

**Interpretation:**

- **Correct prediction** ($\hat{p}_i = 0.99$): Loss = $-\log(0.99) \approx 0.01$ (small loss, good).
- **Uncertain prediction** ($\hat{p}_i = 0.50$): Loss = $-\log(0.50) \approx 0.69$ (moderate loss).
- **Wrong prediction** ($\hat{p}_i = 0.01$): Loss = $-\log(0.01) \approx 4.61$ (large loss, penalized heavily).

**Why Is Logarithm Important?**

The logarithm creates **asymmetric penalties**:
- Small errors (0.99 → 0.95) are forgiven lightly.
- Large errors (0.99 → 0.01) are punished severely.

This is **better than squared error** for classification because it reflects the nature of classification: confidence matters.

**Information-Theoretic Interpretation:**

Cross-entropy is defined as:
$$H(P, Q) = -\sum_i P(i) \log Q(i)$$

It measures the expected information (in bits) needed to describe the true distribution $P$ using a model $Q$.

Minimizing cross-entropy is equivalent to minimizing the **KL divergence** from $P$ (true) to $Q$ (model), i.e., making the model distribution as close to the true distribution as possible.

**Binary Cross-Entropy (Special Case):**

For binary classification ($K = 2$) with true label $y \in \{0, 1\}$ and predicted probability $\hat{p} = \sigma(\theta^T x)$:
$$L_C = -[y \log \hat{p} + (1-y) \log(1 - \hat{p})]$$

If $y = 1$: $L_C = -\log \hat{p}$ (penalize if low probability for class 1).
If $y = 0$: $L_C = -\log(1 - \hat{p})$ (penalize if high probability for class 1).

**Analysis of Incorrect Options:**

- **A)** $L_C = \|h_\theta(x) - y\|_2^2$ is **Squared Error Loss**, used for **regression** (predicting continuous values like house prices). For classification, it treats all misclassifications equally, which is suboptimal. The logarithm in cross-entropy creates better gradients for classification.
- **C)** $L_C = \sigma(\theta^T x)$ is the **output** of logistic regression (a probability), not a loss function.
- **D)** $L_C = \max(0, 1 - y h_\theta(x))$ is **Hinge Loss**, used by Support Vector Machines (SVMs). It focuses on a margin around the decision boundary, not on probability estimates.

**Key Insight:** Cross-entropy is fundamental to modern deep learning. Every time you see classification in practice, cross-entropy loss is likely being used.

## Question Set 6: Generalization and the Bias-Variance Tradeoff

### Q10 (1 point)

**Which of the following best describes the goal of studying generalization in Machine Learning?**

- [ ] A) To minimize the loss on the training set $D_{\text{train}}$ only.
- [x] B) To understand how a model $\hat{h}^*(x)$ trained on $D_{\text{train}}$ will perform on unseen data $D_{\text{test}}$.
- [ ] C) To ensure the variance of the model is always zero.
- [ ] D) To find a transformation $\phi(x)$ that makes the training error zero.

**Answer:** B

**Answer Explanation:**

This question captures the **central goal of machine learning**: **generalization**—the ability to perform well on new, unseen data.

**The Core Problem:**

Machine learning is not about fitting training data; it's about learning patterns that **transfer to the real world**.

Consider a student:
- **Memorization (Bad):** Memorize every answer on a practice test. Score 100% on that test but 20% on the actual exam (overfitting).
- **Understanding (Good):** Learn the underlying concepts. Score 85% on practice test but 80% on exam (good generalization).

The student who generalizes better is the one who **learns principles**, not rote answers.

**Formal Definition:**

Let:
- $D_{\text{train}} = \{(x_i, y_i)\}_{i=1}^{n}$: Training data.
- $D_{\text{test}}$: Unseen test data (same distribution as $D_{\text{train}}$ but different samples).

**Generalization error** is the expected loss on $D_{\text{test}}$:
$$R_{\text{test}}(\hat{h}^*) = \mathbb{E}_{(x,y) \sim D_{\text{test}}} [L(\hat{h}^*(x), y)]$$

**Generalization gap** is the difference:
$$\text{Gap} = R_{\text{test}}(\hat{h}^*) - R_{\text{train}}(\hat{h}^*)$$

**Goal:** Minimize $R_{\text{test}}$ (or equivalently, minimize the gap).

**The Bias-Variance Trade-off (Fundamental):**

When you analyze the expected test error, it decomposes into three parts:
$$\mathbb{E}[\text{Error}_{\text{test}}] = \text{Bias}^2 + \text{Variance} + \text{Irreducible Noise}$$

**Bias² (Underfitting):**
$$\text{Bias} = \mathbb{E}_D[\hat{h}(x)] - h^*(x)$$

The systematic error from using a **too-simple** model.

Example: Fitting a straight line to a quadratic curve always leaves systematic residuals. No amount of data fixes this.

**Variance (Overfitting):**
$$\text{Variance} = \mathbb{E}_D[(\hat{h}(x) - \mathbb{E}_D[\hat{h}(x)])^2]$$

How much the learned model **changes** when you use a different training dataset.

Example: A high-degree polynomial (degree 100) wiggles wildly if you change one data point.

**Irreducible Noise:**

The universe is noisy. Even a perfect model cannot do better than this baseline.

**Visual Understanding:**

```
Error on test set:
                    └─ Best model (Goldilocks zone)
                   /│\
                  / │ \
                 /  │  \
            Low Bias,  \
            High Var    └─ High Bias, Low Var
            (Overfit)        (Underfit)
```

As model complexity increases:
- Bias **decreases** (can fit more complex patterns).
- Variance **increases** (more sensitive to training data noise).
- Test error is **U-shaped**: first decreases (bias dominates), then increases (variance dominates).

**Practical Implications:**

1. **Minimize training error alone = disaster.** It leads to overfitting.
2. **Minimize test error = goal.** This requires balancing bias and variance.
3. **Use validation set:** Hold out part of training data to estimate test error, then tune model complexity.
4. **Regularization:** Add penalties (like L2 regularization: $\lambda \|\theta\|_2^2$) to reduce variance.
5. **Data augmentation:** More training data reduces variance naturally.

**Modern Pragmatic Tip:** Start with a complex model (high variance) and regularize it down. Often more effective than starting simple and hoping to add complexity.

**Analysis of Incorrect Options:**

- **A)** "Minimize the loss on $D_{\text{train}}$ only": This is **overfitting**. A model with 0% training error usually fails on test data. This is exactly what we try to avoid.
- **C)** "Ensure variance is always zero": A model with zero variance is **constant** (always predicts the same value). It refuses to learn. This is pure bias. We want **low** variance, not zero.
- **D)** "Find $\phi(x)$ that makes training error zero": Making training error zero by increasing model complexity is the path to overfitting. We care about **test** error, not training error.

**Key Insight:** The bias-variance tradeoff is the **fundamental principle** of machine learning. It explains why models fail, why data helps, why regularization works, and how to make intelligent model selection decisions.

**Related Topics:** [Bias–Variance Decomposition](notes.md#bias-variance-decomposition)
